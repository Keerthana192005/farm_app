## 🔧 Pre-Deployment Setup Required

### Step 1: Create Render Account
1. Go to [render.com](https://render.com)
2. Sign up with GitHub (recommended)
3. Create new account if needed

### Step 2: Create PostgreSQL Database
1. In Render dashboard → New +
2. Select "PostgreSQL"
3. Create database instance
4. Copy the internal database URL

### Step 3: Deploy the App
1. In Render dashboard → New +
2. Select "Web Service"
3. Connect GitHub repository
4. Configure:
   - **Name**: farm-app (or your choice)
   - **Environment**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn --bind 0.0.0.0:$PORT wsgi:app`
5. Add Environment Variables (from render.yaml):
   - `FLASK_ENV`: production
   - `DATABASE_URL`: (paste the PostgreSQL URL from Step 2)
   - `SECRET_KEY`: (Render generates this automatically)

### Step 4: Update Default Credentials
⚠️ **IMPORTANT**: Change default admin password immediately after deployment!
- Login with: `admin` / `admin123`
- Go to `/admin/change_password`
- Set a strong password

## ⚠️ Known Limitations & Tips

### Free Tier Limitations
- **Spins down after inactivity**: App sleeps after 15 min of no traffic
- **Ephemeral storage**: Files uploaded disappear on restart
- **Limited CPU/RAM**: Suitable for moderate traffic

### Recommendations
1. **Use a file storage service** (AWS S3, Cloudinary) for uploads
2. **Monitor performance** in Render dashboard
3. **Add logging** for debugging in production
4. **Set up alerts** for high CPU/memory usage
5. **Schedule maintenance** during low-traffic hours

## 📞 Support & Troubleshooting

### Common Issues & Solutions

**Issue: "Application failed to start"**
- Check: Database URL is correct in environment variables
- Check: All required packages in requirements.txt
- Check: render.yaml syntax is valid

**Issue: "Port binding error"**
- Solution: ensure START command uses `$PORT` variable
- Check: No hardcoded port 5000 in wsgi.py

**Issue: "Static files not loading"**
- Solution: Flask serves static/ folder automatically
- Check: CSS/JS files are in static/ directory

**Issue: "Database connection error"**
- Solution: Verify DATABASE_URL in environment variables
- Check: Character special encoding in password

**Issue: "Out of disk space"**
- Solution: Delete old database backups
- Check: No massive image uploads in database
